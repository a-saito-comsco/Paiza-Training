ここからは、C#の基礎を学習していただきます。
外部学習サイト「paiza」を活用し、文法の基礎や実装力を養いながら、**Gitを使ってコードをバージョン管理・共有**していきます。

以下の「C#入門編」のうち、「4. 配列」までの演習問題を完了してください。

▶︎ [https://paiza.jp/works/cs/primer](https://paiza.jp/works/cs/primer)

### ステップ②：スキルチェック問題への挑戦

続いて、以下の「スキルチェック問題」に取り組んでください。

 **5問以上の提出** を目指してください。取り組み期間の目安は **2週間** です（進捗に応じて調整します）。
Cランク以下のクリア課題数は不問とします。

▶︎ [https://paiza.jp/challenges/info](https://paiza.jp/challenges/info)

## Git リポジトリの利用ルール

### ① リポジトリ構成（ローカル → push）

Gitリポジトリは個別に用意します。
初期構成は以下のようにしてください：

```
paiza-training/
├── 02_スキルチェック一覧.xlsx
└── skill_check/         # スキルチェックのコード
    ├── B07/
    │   ├── solution_1.cs
    │   └── solution_2.cs
    └── D02/
        └── solution_1.cs
```

* Git基礎でも記載してもらったように`01_学習ログ.xlsx`には、取り組みの内容や日付、振り返りなどを簡単にメモしてください。
* `skill_check/`配下は **(ランク)(問題番号)/solution_1.cs** のように問題ごとにフォルダを分けます。
* 再チャレンジする場合は、`solution_2.cs`のようにファイル名でバージョンを分けてください。

---

### ② Git運用ルール

* 1問ごとにコミットするようにしてください。

* コミットメッセージは簡潔に内容を記述します。
  　例：`Add B07 first solution` / `0731取組課題の追加`

* 毎日作業が終わったら、GitHub（または指定されたリモートリポジトリ）にpushしてください。

* Push後、こちらでコードに対して**レビューコメントを返す**場合があります。

---

### ③ 提出の進捗確認

* push の履歴により進捗確認を行います。
* `02_スキルチェック一覧.xlsx` に取り組んだ課題一覧を記録してください。
